Difference between var, let and const

Outputs can be seen in console (press F12).

Example 1:

        for (var i = 0; i < 3; i++) {
        console.log(i);
        }
        console.log(i);

Inference: The variable i is accessible outside of the for-loop.
This is expected since variables defined with var are only inaccessible outside of a function definition.


Example 2:

        var foo = 1;
        foo = 30;
        var foo = 101;
        console.log(foo);

Inference: Using var you can reassign it with a new value.


Example 3:

        for (let i = 0; i < 3; i++) {
        console.log(i);
        }
        console.log(i);

Inference: let creates a variable which is available only inside the block in which it is defined.


Example 4:

        let foo = 1;
        foo = 30;
        // let foo = 101; Ô∏è // can't use "let" keyword again
        foo = 101;
        console.log(foo);

Inference: let creates a variable which is available only inside the block in which it is defined.


Example 5:

        const myBoolean = true;
        if (myBoolean) {
        const turtles = [
        'leonardo',
        'donatello',
        'michaelangelo',
        'raphael'
        ];
        // turtles = turtles.concat('Shredder'); // this would throw an error
        console.log(turtles);
        }
        console.log(turtles);

Inference: Items can still be added to a const variable that points to an array or an object, because the const is not being reassigned in such case.
It is available only in the block in which it is defined.


Keyword           Scope                       Redefinable?    
var function-scope Yes
let block-scope Yes
const block-scope No

Function-scope: Available through out the function.

Block-scope: Available only within the block in which it is defined.
